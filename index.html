叶芳俊拥有"城市轨道交通供配电技术"与"计算机科学与技术"的跨学科背景。这使他既具备强电系统、继电保护与牵引供电等领域的工程直觉，又掌握了 Python 生态、Linux 系统与边缘计算等弱电侧的开发能力。他致力于将扎实的系统底层逻辑应用于智能硬件开发、网络架构优化及独立软件项目的研发中，期望在"系统稳定"与"快速迭代"之间找到一条可持续的中间路径——既不盲目追新，也不保守停滞，而是通过严谨的设计与可验证的测试，逐步沉淀出可被时间检验的工程产物。

近年来，他以独立开发者的身份参与并主导了多个涉及边缘计算、流媒体管理及数据安全的开源与私有化项目，包括但不限于基于 Cloudflare 全家桶的 FangPress 博客系统、NasMusic-Ai 音乐管理平台、以及强调全加密搜索的 LocalKeyBox 密码库。同时，他在开源社区、服务器自托管、Android 底层系统定制（如 Root 环境管理）等极客领域也保持着长期的实践与探索。

在其个人知识体系中，开源生态与底层原理被视为一切工程的基础。他更倾向于从操作系统内核、网络协议与硬件工作原理切入理解技术，而将框架 API 的使用视为这一理解的派生结果。基于这一立场，他早期对家用电器与廉价电子设备的"随性拆解"被其视为"对系统最朴素的好奇心训练"；而其后续在边缘计算、自托管、嵌入式与 AI 工具链上的工程实践，亦可视作这一朴素好奇心在更高复杂度场景下的自然延伸。

早期经历与教育

早年生活与中学阶段

叶芳俊出生于中国安徽省池州市东至县。初中二年级时，他拥有了第一部真正属于自己的手机——酷派 8712，并开始接触 Android 系统的 Root 与刷机文化，尝试通过修改系统参数更改设备型号等底层信息。2018 年 5 月 19 日（初三学年末），他注册了人生第一个个人域名 yedad.top，并沿用至今。

高中阶段，因课业压力有限，叶芳俊接手了姐姐购入的一台二手笔记本电脑，并以此为契机，逐步自学了系统重装、第三方 Recovery 刷入、ROM 定制以及简单的网页制作等技能。这些早期的"折腾式"实践，使他较早地建立了对计算机软硬件、操作系统及互联网生态的直观认识。

2021 年，叶芳俊参加全国普通高等学校招生全国统一考试。受高考分数及志愿填报结果影响，他被合肥职业技术学院城市轨道交通供配电技术专业录取。该专业属强电方向，主修内容包括高低压电气设备检修、牵引供电系统及底层控制原理。在系统完成本专业课程的同时，他亦完成了相关职业技能培训，并考取了相应的职业资格证书。

进入专科后的大一上学期，叶芳俊即下定决心通过专升本途径转入计算机领域。这一时期，他以C 语言为起点开始了系统性的编程自学，为后续本科阶段的学习与项目研发奠定了基础。

本科阶段（2024–2026）

2024 年，叶芳俊通过安徽省专升本统一考试，被皖江工学院计算机科学与技术专业录取，开始全日制本科阶段的学习，于 2026 年 6 月 24 日 毕业。

本科期间，叶芳俊完成了从硬件电气向软件工程的系统性转型。其学习与研究方向逐步聚焦于 Python（Django / FastAPI）后端开发、Vue.js 前端应用构建以及 Linux 系统的运维与底层逻辑，为其日后从事软件测试、IT 技术支持及运维工作奠定了技术基础。

核心工程项目

FangPress 极简博客系统

FangPress 是一套强调“零服务器、零成本、零运维”的独立博客系统。该项目前端采用纯 HTML + 原生 JS + Tailwind CSS (v4) 构建并实现三端响应式适配，后端逻辑完全依赖于 Cloudflare 边缘网络生态，包括 Pages Functions (Workers)、D1 (托管 SQLite)、R2 (对象存储) 以及 KV (边缘缓存)。

该系统主打“纯文字内容流”，支持 Markdown 文章与轻量级推文的双轨发布（自动分配 Slug 标签）。其管理后台支持可视化增删改查及站点导航的 SSR 即时生效，并创新性地支持在本地 PHP 环境直接发布或修改文章。此外，FangPress 深度整合了 AI 写作工作流，内置提示词（Prompt），可直接引导大语言模型生成符合系统数据库结构的文章 JSON 数据。

NasMusic-Ai 本地音乐库管理系统

基于 Django + Vue 3 + Gemini 构建的现代化本地音乐管理平台。该系统可作为高效流媒体服务配合 Nginx 部署，为用户提供沉浸式的私人流媒体体验。核心功能涵盖：

  • 智能入库与刮削：自动扫描本地目录，读取并提取音频文件的 ID3 及 Vorbis 标签，支持从网络单首或批量自动刮削封面与歌词，并具备多歌手智能拆分功能。
  • 元数据同步：在前端在线编辑歌曲标题、专辑、封面等信息后，系统自动同步并将修改物理写入音频文件的底层标签中。
  • 高级文件管理：支持大文件分片上传，并内置完整的“回收站”逻辑，被删除的文件移入 .trash 目录以供恢复或安全销毁。
  • 时间轴打点：内置可视化的 LRC 歌词编辑模块，支持键盘快捷键精确定位时间轴，并同步写入音频文件。

LocalKeyBox 本地密码库

LocalKeyBox 是一项注重本地数据主权与安全性的密码库项目。该工具后端基于 FastAPI 构建，前端采用 Vue 3 + Vite 提供现代化的响应式用户界面。其核心技术突破在于实现了基于 SQLite 的全加密盲引搜索功能，配合 JWT 安全认证机制，确保敏感数据在本地完全受控且以密文形式存储与检索，杜绝了中心化云端泄露的风险。

LRC 歌词在线编辑器

作为流媒体生态的衍生微服务，该在线工具（托管于 lrcedit.yedad.top）为用户提供了一个直观的 Web 端 LRC 歌词编辑环境，支持可视化时间轴调节与歌词校对，并允许用户一键导出标准化格式的 LRC 文件，广泛服务于本地音乐整理与卡拉OK制作需求。

医院排班调度系统

作为本科阶段的重要毕业实践工程，该动态调度系统采用了 Django 框架提供稳定高并发的后端接口，前端基于 Vue 3 渲染。项目实现了复杂的科室人员轮转逻辑，验证了其在 Python 基础开发及软件结构化设计方面的能力。

微信生态：工具类小程序与公众号

除基于 Web 的独立应用外，叶芳俊亦在微信生态内开展了系统性的开发与运营实践，其作品定位为面向特定场景的工具类小程序与工具类公众号。鉴于运营策略与隐私考虑，相关产品名称与具体内容暂未在公开渠道披露。

小程序方面，其作品以工具属性为核心定位，使用微信官方开发者工具及原生 WXML / WXSS / JavaScript 技术栈完成开发，并已正式上线提供服务。该项目使其熟悉了微信小程序的应用框架、云开发能力及提审发布流程。

公众号方面，他同时运营着一个工具类定位的微信公众号，定期发布原创技术文章。该账号与其小程序相互独立、暂无业务联动，定位为内容输出与长期沉淀的载体。

研究与技术偏好

有别于对各类主流框架的广泛覆盖式学习，叶芳俊在技术路径上倾向于在少数技术栈上作纵向下钻，以系统稳定性、数据主权与长期可维护性为优先目标。其研究与工程实践覆盖以下层面。

选型原则

在长期实践中，叶芳俊形成了以下几条正面技术偏好：

  • 轻量优先：优先选择部署与运维成本低、依赖面窄的方案，典型表现为以 SQLite 替代大型关系数据库、以原生 HTML / JavaScript 替代重型 SPA 框架。
  • 数据主权：在涉及敏感数据的项目中坚持本地优先、加密存储与可控部署；该原则在 LocalKeyBox 中体现为基于 SQLite 的全加密盲引搜索。
  • 底层下钻：相对框架表层 API，更重视操作系统、网络协议与硬件工作原理；该倾向在 EZpad 4s 改造 等硬件级项目中得到直接体现。
  • 开源与社区活跃度：优先选择开源、社区维护活跃、可长期演进的生态，避免被单一商业方锁定。

主要研究方向

后端与数据层

主语言为 Python，熟练使用 Django 与 FastAPI 构建 REST 风格接口。数据层长期以 SQLite 为首选关系型存储，关注 SQL 与索引设计、事务约束与连接池策略对数据完整性与查询性能的影响；该方向直接服务于 NasMusic-Ai 的元数据检索与 LocalKeyBox 的加密索引场景。

前端与界面工程化

在前端方向，倾向于使用 Vue 3 配合 Tailwind CSS 构建响应式界面，强调组件化拆分、可维护的样式结构与 SSR（服务端渲染）能力；FangPress 管理后台即通过 Vue 3 实现导航项的"修改即生效"。

边缘计算与 Serverless 生态

长期使用 Cloudflare 边缘生态构建无服务器应用，具体涉及 Pages（静态托管 + Functions）、Workers（边缘函数）、D1（托管 SQLite）、KV（边缘键值缓存）与 R2（对象存储）。该方向在 FangPress 系统中体现为"零服务器、零成本、零运维"的整体架构。

Linux 与嵌入式软硬件

在系统层面，深度使用 Ubuntu、Armbian（ARM 设备 Linux 发行版）与 OpenWrt（路由器固件）等系统。在硬件调试方面，常用工具有烙铁、万用表与 示波器，并熟悉 ESP32 系列微控制器的开发与调试；相关能力支撑了从路由器、智能家居到自制 mini 服务器的多种 DIY 场景。

软件测试

在测试方向上，其工具链涵盖：单元测试（pytest 等）、接口测试（Postman、Requests 等）、性能测试（Locust 等）以及基于 OWASP 框架的基础安全扫描。其工作流强调在小型个人项目中亦引入自动化测试与持续集成，以降低回归成本。

AI 本地化与工具链集成

密切关注 Gemini 等大语言模型的能力演进，主要探索方向为提示词工程（Prompt Engineering）与 AI 工作流集成。代表实践为 FangPress 系统内置的 AI 写作助手——通过预设的提示词模板，可直接引导大语言模型生成符合该系统数据库结构（JSON）的文章草稿，将 AI 能力嵌入到内容生产链路中。

动手实践与自托管

自幼起，叶芳俊便对各类电器与电子设备的内部结构抱有强烈好奇心，习惯于"随性拆解"以了解其工作原理。进入中学后，他将这一兴趣延伸至计算机与网络设备领域，先后接触并研究过家用路由器、NAS（网络附加存储）以及软路由等多种设备。

在家庭网络环境中，他搭建了一台小型的内网服务器（4 GB 内存 + 64 GB 存储），主要用于文件存储与轻量级服务部署，并先后在该设备上运行了 NasMusic-Ai 流媒体平台与 LocalKeyBox 本地密码库等自有项目。

此外，叶芳俊曾在二手交易平台购入一台 EZpad 4s 平板电脑，并将其改装为一台 mini Ubuntu 服务器。改造过程中，他通过以下步骤完成硬件重构：

  • 拆除原机电池后，利用万用表对电源管理芯片进行测量，发现特定触点需约 20 Ω 接地电阻方能模拟电池温度信息并正常开机，遂手工焊上相应电阻；
  • 利用机身 USB 接口的焊点接入外置 USB Hub，再借助 Hub 的供电触点引出 USB 网卡，实现了平板的有线网络接入功能。

此次改造是其在硬件拆解、电子焊接与底层系统调试方面的代表性实践之一。

智能家居与嵌入式

智能家居是叶芳俊长期关注的另一方向。鉴于在校期间宿舍环境的限制，相关实践尚处于入门阶段。他于近期购入了一块 ESP32-S3 开发板，并基于其实现了"Wi-Fi USB 键盘"功能——设备通过 Wi-Fi 连接后，可被主机识别为一枚标准 USB 键盘，从而支持远程输入与轻量自动化控制。他表示，待未来拥有独立居所后，将系统性地推进家庭自动化方案的落地。
